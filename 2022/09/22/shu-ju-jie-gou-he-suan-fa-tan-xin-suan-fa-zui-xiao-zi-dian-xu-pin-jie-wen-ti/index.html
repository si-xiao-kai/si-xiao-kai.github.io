1.概述

1)贪心算法是最符合自然智慧的算法。

2)贪心算法用一种局部最功利的标准，总是做出在当前看来是最好的选择。

3)难点在于证明局部最功利的标准可以得到全局最优解，【怎么证明贪心策略是否有效】。

4)有些问题用贪心策略能够获得最优解，有些则不然。

5)对于贪心算法的学习主要以增加阅历和经验为主。

贪心算法常用的结构：

  小根堆、大根堆、排序是贪心算法最常用的手段。

  堆天然可以根据自定义的排序标准排列数据，然后依次弹出想要的数据。

  贪心策略的代码量很少：贪心策略+堆（和排序）

2.能用贪心策略求解的例子

假设一个数组中$N$个正数

定义：分数score=第一个挑出来的数*1+第二个挑出来的数*2+第三个挑出来的数*3+….

目标获得最大分数。

有效贪心策略：数组中小样本✖️小系数，然后累加

基本想法：从小到大排序，从前往后逐个挑出来。

很显然，这是一个有效的贪心策略。

4.贪心算法经典例子（拼接字典序最小的字符串）

  给定一个由字符串组成的数组strs,必须把所有的字符串拼接起来,返回所有可能的拼接结果中，字典序最小的结果。

4.1 字典序

（1）如果两个字符串长度一样，直接按照k进制的数进行比较（k取决于字符集大小，例如全是英文字母，可以视作26进制数）。

（2）如果两个字符串长度不一样，先补“0”到长度一致，然后按照（1）比较，这里的0代指k进制数的最小的数字。

例如:ba 与 b，b < ba

例如:mk 与 ts , mk < ts

注：Java中的字符串比较方法就是在比较字典序

4.2 直观（错误）想法

  将数组中的字符串，按照字典序排序，然后逐个拼接。

反例:

字符数组: [“ba”,”b”]

排序: “b” “ba”

拼接: “bba”

但是还有字典序更小的结果

“bab”=”ba”.”b” < “bba”

上面的反例表明上述排序方案构造的贪心策略得不到正确的最优解。

4.4 证明排序方案2能够获得最优解

4.4.1 证明排序方案2的排序策略具有传递性

  有些自定义的排序策略不一定具有传递性，a ≤ b,b ≤ c，推不出 a ≤ c 则不具备传递性，没有传递性的排序策略是没有意义的。

4.4.2 证明按照安排序方案2排序之后，拼接的字符串的字典序最小

  证明任意调换2个字符串，字典序会变大

证明任意调换2个字符串，字典序会变大

  同理证明任意调换3个字符串，字典序会变大

  然后利用数学归纳法，证明任意改动排列的顺序，字典序都会变大，那么即可证明按照安排序方案2排序之后，拼接的字符串的字典序最小。

2）strsFullPermutation（）方法返回字符串数组中字符串拼接的全排列（full permutation）（递归）

    public static TreeSet<String> strsFullPermutation(String[] strs) {
        TreeSet<String> resSet=new TreeSet<>();
        //如果传入空数组，返回空字符串的resSet TreeSet;
        if(strs.length==0) {
            resSet.add("");
            return resSet;
        }
        //如果传入的字符串数组不为空，那么递归返回全排列
        //
        for(int i=0;i<strs.length;i++) {
            String first=strs[i];
            //返回除去first之后剩余的字符串数组
            String[] restStrs=removeIndexstrFromStrs(strs, i);
            TreeSet<String> next=strsFullPermutation(restStrs);//剩余字符串数组的全排列
            for(String s:next) {
                resSet.add(first+s);
            }
        }
        return resSet;
    }

3）removeIndexstrFromStrs（）方法从字符串数组中删除一个字符串返回剩下的字符串数组

    //功能函数，removeIndexstrFromStrs从字符串数组中删除一个字符串返回剩下的字符串数组。*
    public static String[] removeIndexstrFromStrs(String[] strs,int index) {
        String[] tmp=new String[strs.length-1];
        int j=0;//关键再给一个变量记录新数组的下标变化
//        for(int i=0;i<strs.length;i++) {
//            if(i==index)
//            {
//                continue;
//            }
//            tmp[j++]=strs[i];    
//        }
        //简化
        for(int i=0;i<strs.length;i++) {
            if(i!=index) {
                tmp[j++]=strs[i];    
            }
                
        }
        return tmp;
    }

4.5.2 自定义贪心策略实现返回字符串数组拼接最小字典序的方法

1）自定义比较器

    public static class MyComparator implements Comparator<String>{
        @Override
        public int compare(String s1,String s2) {
            return (s1+s2).compareTo(s2+s1);
        }
    }//s1和s2,如果s1拼接s2的字典序小于等于s2拼接s1的字典序，则s1排在s2的前面

2）通过自定义的排序策略对字符串数组进行排序，获得最点序最小的拼接字符串

    public static String LowestLexicographyString02(String[] strs) {
        if(strs==null||strs.length==0) {
            return "";
        }
        Arrays.sort(strs,new MyComparator());
        String ans="";
        for(int i=0;i<strs.length;i++) {
            ans+=strs[i];
        }
        return ans;
    }

4.5.3 编写对数器测试方法的正确性

1）随机生成字符串数组生成函数

    public static String[] generateRandomStringArray(int arrlength,int strlength) {
        String[] strs=new String[(int)(arrlength*Math.random()+1)];//[1,arrlength]随机长度
        for(int i=0;i<strs.length;i++) {
            strs[i]=generateRandomString(strlength);
        }
        return strs;
    }

2）随机字符串生成函数

    public static String generateRandomString(int strlen) {
        char[] res=new char[(int)(Math.random()*strlen+1)];//[1,strlen]随机长度的字符数组
        String s="";
        for(int i=0;i<res.length;i++) {
            res[i]=(char)(Math.random()*25+97); //97-a 122-z //[97,122]中的随机数
            s+=res[i];
        }
        return s;
    }

3）字符串数组拷贝函数

    public static String[] copyStringArray(String[] strs) {
        String[] newstrStrings=new String[strs.length];
        for(int i=0;i<strs.length;i++) {
            newstrStrings[i]=strs[i];
        }
        return newstrStrings;
    }
